udplink

面向嵌入式与 MCU 场景的可靠 UDP 库。当前仓库已经打通核心协议、最小 C ABI、安装导出与 Windows 主线验证，但 C ABI 仍保持 wait 状态，PlatformIO 仍是 Arduino-first 的窄路径。

GitHub Project Status

当前重点

如果你只想快速判断仓库现状：Windows 验证链最完整，C ABI 已实现但尚未声明长期稳定，板级 bring-up 文档已准备好但仍缺真实硬件证据。

已验证内容

  • C++ 核心协议栈可在 Windows / Linux / macOS 构建与测试。
  • 最小 C ABI 已进入测试、安装导出与维护示例路径。
  • Windows 已验证 ctest、install-consume、C API consumer example、release check。
  • host-side rudp_example_udp_peer 已可构建，用于后续板级 bring-up。

当前约束

  • C ABI 仍处于 wait，还没有被正式声明长期稳定。
  • examples/c_api/install_consume/ 还没有被维护者正式提升为首个受支持 downstream baseline。
  • PlatformIO 仍是 Arduino-first 窄路径，不代表广泛嵌入式支持承诺。

下一步最有价值的方向

  • 保持 Windows build/test/install-consume 持续绿色。
  • 继续收敛 README、首页、release checklist、维护文档之间的重复与漂移。
  • 获取一次真实 Arduino Mega 2560 + W5100 板级证据运行。
  • 之后再决定 C ABI 是继续 wait 还是进入 stable。

快速开始

# clone
git clone https://github.com/KairuiKin/udplink.git
cd udplink

# build
cmake -S . -B build
cmake --build build --config Release

# test on Unix-like
ctest --test-dir build --output-on-failure

# test on Windows PowerShell
ctest --test-dir build -C Release --output-on-failure

Minimal Endpoint Snippet

This snippet is synced from examples/basic_endpoint.cpp by python scripts/sync_docs_snippets.py.

rudp::Config cfg = rudp::ConfigForProfile(rudp::ConfigProfile::kBalanced);
cfg.enable_auth = true;
cfg.auth_key0 = 0x0706050403020100ull;
cfg.auth_key1 = 0x0F0E0D0C0B0A0908ull;

rudp::Endpoint a;
rudp::Endpoint b;
rudp::Hooks ha = {&wire, NowMs, SendA, 0, DeliverA, 0, 0};
rudp::Hooks hb = {&wire, NowMs, SendB, 0, DeliverB, 0, 0};
if (!a.Init(cfg, ha) || !b.Init(cfg, hb)) return 1;
a.StartConnect();

验证路径

  • rudp_self_test: smoke path for connect/queue/timeout behavior.
  • rudp_reliability_test: deterministic loss / reorder regression.
  • rudp_manager_test: multi-connection manager lifecycle and query path.
  • rudp_c_api_test: minimal C ABI lifecycle, config, and query smoke.
  • tests/install_consume: installed-package C++ consumer validation.
  • tests/install_consume_c: installed-package C consumer validation.
  • examples/c_api/install_consume: maintained readable C consumer example.
  • python scripts/release_check.py: release-style local validation chain.

Windows 完成定义

  • Main project Release build succeeds.
  • ctest --test-dir build -C Release --output-on-failure passes.
  • Install-consume paths configure, build, and run successfully.
  • Windows consumers prefer rudp_DIR:PATH=<stage>\lib\cmake\rudp.
